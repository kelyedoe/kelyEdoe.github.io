INSTALLATION DE LARAVEL SUR WINDOWS

Tuto 01

Dans ce tutoriel nous allons commencer par installer laravel sur Windows. Avant de nous mettre à fond dans le vif du sujet, il est important de savoir qu'il est souvent difficile de déployer en production une application PHP/Laravel sur un serveur windows. Cela est dû au fait que la majorité des outils web sont plus adaptés aux environnements Unix, Mac, Linux.

Les développeurs souhaitent ainsi donc avoir un environnement de développement qui est proche de l'environnement de production pour des raisons de fiabilité. Il existe une panoplie de façon de faire pour installer laravel sur windows mais comme d'habitude je vous montre les bonnes pratiques à adopter. Nous verrons deux façons de faire: Homestead(une machine virtuelle Ubuntu sous windows) et aussi Laragon.

La première méthode est relativement complexe j'avoue mais c'est bien de le connaitre.

C'est quoi Homestead ? (Une machine virtuelle Ubuntu sous windows)

Ce qu'il faudra retenir ici

Homestead est l'outil officiel qui fournit une machine virtuelle entièrement paramétrée pour développer une application laravel en local tout en étant plus proche des environnements de production.

Installer et Initialiser Homestead

1. VirtualBox et Vagrant

VirtualBox: est un logiciel de virtualisation libre distribué par oracle et nécessaire à l'installation de Homestead. Lien pour télécharger: Lien de téléchargement de Virtual Box Ici...

Vagrant: C'est une application utilisée pour la création des environnements de développement. Ce logiciel se base sur la virtualBox pour l'installation de la box Homestead. Lien pour télécharger Vagrant: Lien de téléchargement de Vagrant ici...

2. Le terminal

Le terminal c'est un outil qui vous permet d'envoyer des instructions au système d'exploitation en utilisant des lignes de commande. Ex: PowerShell Le PowerShell ainsi l'invite de commande de windows peuvent être limitée, les devs recommandent Cmder qui est une alternative pour la programmation sur windows. Lien pour telecharger Cmder: cliquez ici pour telecharger

3. Git

Git est un outil de versionning qui est nécessaire pour installer Homestead. Lien pour telecharger git: https://git-scm.com/downloads

4. Ajouter la box de Homestead

Accéder à votre terminal en tant qu'admin et taper la commande suivante pour ajouter la box Homestead à Vagrant.

vagrant box add laravel/homestead

5. Recuperation du depot de Homestead

Nous allons maintenant récupérer le dépôt Homestead depuis Github dans le dossier utilisateur

C: cd C:\Users\VotreNom git clone https://github.com/laravel/homestead.git Homestead

Accédez au dossier Homestead qu'on vient de créer et lancer le script d'initialisation suivant

cd Homestead init.bat

7. Créer une clé SSH

La clé SSH est importante pour utiliser SSH. Pour vérifier son existence:

  • veuillez accéder au dossier C:\Users\votreNom
  • Rechercher le dossier .ssh (possible qu'il soit caché)
  • vérifier à l'intérieur de ce dossier les deux fichiers suivants : id_rsa, id_rsa.pub.

Bien sûr au cas vous ne l'avez pas sur votre machine, voici comment l'installer:

  • ouvrez votre terminal (cmder ou git bash ...)
  • exécutez la commande suivante:

ssh-keygen -t rsa "votre_email@gmail.com"

  • Appuyer sur entrée à chaque fois pour laisser les paramètres par défaut.

8. Créer un dossier de travail

Ce dossier peut créer n'importe ou. Il est cependant primordial de le créer car il sera partagé avec Homestead pour faire fonctionner vos applications laravel. Ps:) Veillez à le créer avant de continuer...

9. Modifier le fichier Homestead.yaml

Cette étape est aussi important car elle vous permet de faire fonctionner le fichier Homestead.yaml

  • Accédez au fichier Homestead.yaml dans le dossier Homestead créé à l'étape 5

L'étape suivante permet d'indiquer le chemin vers la clé SSH. On autorise ainsi la connexion à la machine virtuelle avec le terminal.

  • Remplacer
Authorize: ~/.ssh/id_rsa.pub
keys:
    ~/.ssh/id_rsa
  • Par
Authorize: C:/Users/votreNom/.ssh/id_rsa.pub


keys:
    C:/Users/votreNom/.ssh/id_rsa

Ensuite à travers cette étape on va faire le lien entre le dossier de travail et la machine virtuelle.

  • Remplacer
folders:
    - map: ~/Code
to: /home/vagrant/Code
  • Par
folders:
    -map: C:/Users/votreNom/votre_dossier_de_travail_crée


to:
   /home/vagrant/votre_dossier_de_travail_crée

On devra donc remplir la partie de cette étape à chaque création d'une nouvelle application laravel. Pour commencer nous allons d'abord supprimer toutes les déclaration d'application et des base de données du fichier .yaml

  • Remplacer
sites:
    - map: homestead.app
    to: /home/vagrant/Code/public


databases:
    - homestead
  • Par
sites:
     
databases:

Créer votre application Laravel

Vous pouvez donc créer votre application laravel.

1. Installer composer

Composer est un logiciel gestionnaire libre des dépendances pour PHP. Il vous permet de déclarer et d'installer des bibliothèques dont votre projet a besoin.

Ps: Pour télécharger et installer composer: Cliquez ici ...

2. Telecharger Laravel

Accédez à votre terminal et taper la ligne de commande:

composer global require "laravel/installer

PS: Votre environnement de travail est prêt. Pour vos prochains projets vous n'aurez qu'à répéter les étapes qui suivront.

3. Créer votre application web

Accédez à votre dossier de travail en lancez la commande suivante:

c:
cd c:\Users\votreNom\votre_dossier_de_travail
laravel new nom_du_projet

4. Configuration l'application pour Homestead

Comme susmentionné, vous aurez besoin de déclarer votre application à chaque fois dans le fichier Homestead.yaml. On y ajoutera le nom de l'application et de la base de données spécifiques à notre application.

sites:
    - map: nom_du_projet.test
    to: /home/vagrant/votre_dossier_de_travail/nom_du_projet/public

databases:
    - nom_du_projet

Enfin modifier le fichier hosts de windows pour faire le lien entre l'IP de la machine virtuelle et le nom de domaine déclaré.

Ce fichier se nomme hosts et se trouve dans: *c:\Windows\System32\drivers\etc*

Ajoutez une entrée à la fin de ce fichier hosts: 192.168.10.10 nom_du_projet.test

  • Acceder à l'application laravel PS: Assurez-vous d'avoir lancé votre machine virtuelle.
>   c:
>   cd c:\Users\votre_nom\Homestead
>   vagrant up

  • conclusion

Votre application est maintenant disponible depuis votre navigateur à l'adresse : http://non_du_projet.test
