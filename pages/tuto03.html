Environnement

Les options de configuration dependent de l'environnement. L'accès à la base de données est different sur l'environnement de développement et l'envrironnement de production. Laravel propose un système de variables liées à l'environnement.

En laravel les variables sont stockées dans le fichier .env que vous retrouvez à la racine de votre projet. Ce fichier ne doit pas etre partagés en aucun cas.

Lorsque vous créez un depot git pour votre projjet. Vous allez remarquer que ce fichier .env ne se retrouve pas dans le depot git. Cela se fait grace aux grace aux regles decrites dans le fiichier .gitignore

Artisan

C'est interface en ligne de commande de laravel. Artisan est utiliser frequement dans un projet laravel pour creer un controller, un modele, une migration, etc ...

Pour utiliser, rendez vous dans votre terminal et accerder à la racine de votre application. Utiliser la commande suivante pour afficher la liste des commandes disponible dans artisan.

php artisan list

Quelques commandes pour le debogage

- dd

Cette instruction prends en paramètre une variable ou un objet. Elle permet d'afficher le contenu de la variable passée en paramètre. Elle a cependant la particularité d'arreter tout le programme apres avoir été exécutéé.

- dump

L'instruction dump est similaire à dd mais celle ci n'arrete pas le programme.

w- log
