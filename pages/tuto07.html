Introduction

Les Redirections

Imaginez que dans votre application une route est devenue obselète pour une raison ouu une autre. Au lieu de retourner une erreur 404(Page Introuvable) à votre utilisateur qui n'a que l'ancienne route, il est plus pratique et surtout professionel de le rediriger sur votre nouvelle.

Pour se faire, laravel met à notre disposition la fonction Redirect() qui prend en paramètre l'URI vers laquelle renvoyer.

Code:

    Route::get('ancienneURI', function(){
        return redirect('nouvelleURI');
    });
    Route::get('nouvelleURI', function(){
        return "Votre nouvelle page";
    });

Dans le code ci dessus, un utilisateur qui tape l'ancienne URI: ancienneURI dan sson navigateur est rediriger directement sur la nouvelle URI nouvelleURI.

NB: La fonction redirect() permet également de retourner sur la page précédenment visitée grace à l'instruction:

Code:

    
        return redirect()->back();

Le Groupage des routes

Dans une application, il est fréquent d'avoir plusieurs routes. Il devient alors très pratique de pouvoir les distinguer en utilisant des prefixes.

Par exemple dans une application ou nous avons des routes pour les admins et des routes pour les visiteurs on pourra les distinguer en les écrivant de la facon suivante:

Code:

    // Pour les visiteurs
        Route::get('visiteur/nouveau-compte', function(){...}); 
        Route::get('visiteur/profile', function(){...}); 
        Route::get('visiteur/mes-articles', function(){...}); 

        // Pour les administrateurs
       Route::get('admin/stats', function(){...}); 
        Route::get('admin/users', function(){...});
        Route::get('admin/users/{id}', function($id){...});
        Route::get('admin/articles', function(){...});
        Route::get('admin/'article/{id}', function($id){...});

Vous l'auriez remarqué, les routes des visiteurs sont toutes précedées du prefixe visiteur, de meme les routes des admin sont aussi précédées par admin. Pour éviter cette répétition et avoir une clarté dans notre carte de routes, laravel propose la méthode route::group().

route::Group() permet au developpeur de regrouper les routes ayant un meme prefixe.

Pour l'exemple précédent on pourra s'écrire simplement:

Code:

    // Pour les visiteurs
        Route::group(['prefix'=>'visiteur'], function(){

        Route::get('nouveau-compte', function(){...}); 
        Route::get('mes-articles', function(){...}); 
        Route::get('profile', function(){...});
        Route::get('mes-articles', function(){...});
 }); 
        
        

        // Pour les administrateurs
        Route::group(['prefix'=>'admin'], function(){
            
        Route::get('stats', function(){...}); 
        Route::get('users', function(){...});
        Route::get('users/{id}', function($id){...});
        Route::get('articles', function(){...});
        Route::get('article/{id}', function($id){...});
 });

Vous avez remarquez surement que dans l'exemple précédent on retrouve les meme prefixe à l'intérieur des route groupé sous le préfixe admin. On se retrouve alors face au meme problème qu'au départ.

Heureusement, laravel nous permet de créer des sous-groupes.

Les sous groupes

On pourra alors réecrires les routes pour les admins de la facon suivante. Code:

    
        // Pour les administrateurs
        Route::group(['prefix'=>'admin'], function(){
            
        Route::get('stats', function(){...}); 

        //Sous Groupe avec prefixe : users
        Route::group(['prefix'=> 'users'], function(){
            Route::get('/', function(){...});
            Route::get('{id}', function($id){...});
        });
        //Sous Groupe avec prefixe : articles
        Route::group(['prefix'=> 'articles'], function(){
            Route::get('/', function(){...});
             Route::get('{id}', function($id){...});
        });
 });

Les sous-domaines en laravel

Avec laravel il est possible de mettre en place des applications mutitenants. On appelle application mutitenants une application qui peut etre accessible via plusieurs sous domaines.

Par exemple, imaginez un supermarché qui dispose du non de domaine principal http//green-market.io/ . Notre surper marché green market offre différents services par exemple de restauration et aussi de voyage. Les sites de ces différents services pourraient avoir les sous domaines respectifs :

  • http//resto.green-market.io/
  • http//voyage.green-market.io/

En laravel, il est possible de créer des routes dans lesquelles on peut passer les noms des sous-domaines en paramètre.

Dans notre cas, voici un exemple:

   Route::group (['domain' => '{service}.green-market.io']) {

    Route::get('/', function($service){
        return "Bienvenue chez Green Market";
    });

    Route::get('/nos-horaires', function($service){
        return "Nos horaires sont les suivants:";
    });
});
