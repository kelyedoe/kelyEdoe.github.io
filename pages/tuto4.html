Introduction

Il est très important de définir que ce soit dans votre site ou application web des routes qui ont un sens et qui sont aisément lisible par les utilisateurs. Un route permet d'associer un URI(Identifiant de Ressources Uniformes) à un code spécifique.

Heureusement, Laravel fournit un systéme de route très simple d'usage. La liste des routes d'un projet laravel se retrouve dans le dossier Routes et dans le fichier web.php. A chaque qu'une nouvelle pages ou vues s'ajoute à votre projet il faudra donc modifier ce fichier afin de pouvoir accéder à votre via le navigateur.

Création d'un route

Pour definir une route en laravel, on appelle la classe Route précédée de la méthode HTTP souhaitée. Nous indiquerons alors à cette méthode l'URI et le code qui sera exécuté une fois qu'on accède au lien.

Exemple:

Route::get('accueil', function(){
    return "Bienvenue sur Qk Learn";
});

Ici, quand on accède à la page /accueil du projet dans un navigateur on verra affiché le message "Bienvenue sur Qk Learn".

On remarque que dans ce exemple la méthode HTTP utilisée est GET. Il existe d'autres méthodes HTTP que vous aurez à utiliser selon les situations. Dans le prochains paragraphe je vous présente quelques méthodes HTTP (les plus utilisées).

Quelques méthodes HTTP

Méthodes HTTP Description
GET C'est la méthode la plus courante pour demander une ressource. Une requête GET est sans effet sur la ressource, il doit être possible de répéter la requête sans effet.
POST Cette méthode permet de transférer des données vers la ressource spécifiée. Elle est le plus souvent utilisée dans les formmulaires.
PUT Elle est utilisé pour mettre à jour ou pour remplacer une ressource existente sur le serveur.
DELETE Elle supprime la ressource qui lui sera indiquéé
PATCH Cette méthode nous permet d'apporter des modifications partielle à une ressource existente.
HEAD Cette méthode est utilisé pour retrouver l'entete qui serait affichée si une resource est demandée par la méthode HTTP Get.

NB: Voici les quelques méthodes HTTP les plus utilisées dans les projets web. Cette liste n'est bien sur pas exaustive, je vous invite donc à visiter wikipédia pour en savoir plus au sujet des méthodes HTTP.

La fonction de rappel (Callback)

Encore appelée fonction anonyme, les fonction de rappel ou les Callback sont utilisées pour renoyer une reponse . On l'ecrit généralement dans le fichier web.php . Cette fonction doit contenir l'instruction return afin de retourner quelque chose dans le navigateur.

On peut écrire du code PHP dans une fonction callback.

Code:

Route::get('article/{id}', function($id){
    $article = get_article($id);
    return $article;
});

Dans l'exemple on a une route GET qui accède à un article spécifié par son id via l'URI /article/{id}.

On recupère l'id en paramètre dans la fonction callback et on affiche l'article indiqué dans le navigateur.

Ici on suppose qu'on a deja defini une fonction get_article qui se charge de recuperer l'article ayant l'id spécifié dans la base de données.

On retourne alors l'article dans l'instruction return.
